    <article class="markdown-body entry-content" itemprop="text"><h1><a id="user-content-x2js---xml-to-json-and-vice-versa-for-javascript" class="anchor" href="#x2js---xml-to-json-and-vice-versa-for-javascript" aria-hidden="true"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1h-1c-1.5 0-3-1.69-3-3.5s1.55-3.5 3-3.5h4c1.45 0 3 1.69 3 3.5 0 1.41-0.91 2.72-2 3.25v-1.16c0.58-0.45 1-1.27 1-2.09 0-1.28-1.02-2.5-2-2.5H4c-0.98 0-2 1.22-2 2.5s1 2.5 2 2.5z m9-3h-1v1h1c1 0 2 1.22 2 2.5s-1.02 2.5-2 2.5H9c-0.98 0-2-1.22-2-2.5 0-0.83 0.42-1.64 1-2.09v-1.16c-1.09 0.53-2 1.84-2 3.25 0 1.81 1.55 3.5 3 3.5h4c1.45 0 3-1.69 3-3.5s-1.5-3.5-3-3.5z"></path></svg></a>x2js - XML to JSON and vice versa for JavaScript</h1>

<p>This library provides XML to JSON (JavaScript Objects) and vice versa javascript conversion functions.
The library is very small and has no any dependencies.</p>

<h2><a id="user-content-api-functions" class="anchor" href="#api-functions" aria-hidden="true"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1h-1c-1.5 0-3-1.69-3-3.5s1.55-3.5 3-3.5h4c1.45 0 3 1.69 3 3.5 0 1.41-0.91 2.72-2 3.25v-1.16c0.58-0.45 1-1.27 1-2.09 0-1.28-1.02-2.5-2-2.5H4c-0.98 0-2 1.22-2 2.5s1 2.5 2 2.5z m9-3h-1v1h1c1 0 2 1.22 2 2.5s-1.02 2.5-2 2.5H9c-0.98 0-2-1.22-2-2.5 0-0.83 0.42-1.64 1-2.09v-1.16c-1.09 0.53-2 1.84-2 3.25 0 1.81 1.55 3.5 3 3.5h4c1.45 0 3-1.69 3-3.5s-1.5-3.5-3-3.5z"></path></svg></a>API functions</h2>

<ul>
<li><code>new X2JS()</code> - to create your own instance to access all library functionality</li>
<li><p><code>new X2JS(config)</code> - to create your own instance with additional config</p></li>
<li><p><code>&lt;instance&gt;.xml2json</code> - Convert XML specified as DOM Object to JSON</p></li>
<li><code>&lt;instance&gt;.json2xml</code> - Convert JSON to XML DOM Object</li>
<li><code>&lt;instance&gt;.xml_str2json</code> - Convert XML specified as string to JSON</li>
<li><code>&lt;instance&gt;.json2xml_str</code> - Convert JSON to XML string</li>
<li><code>&lt;instance&gt;.asArray</code> - Utility function to work with a JSON field always in array form</li>
<li><code>&lt;instance&gt;.asDateTime</code> - Utility function to convert the specified parameter from XML DateTime to JS Date</li>
<li><code>&lt;instance&gt;.asXmlDateTime</code> - Utility function to convert the specified parameter to XML DateTime from JS Date or timestamp</li>
</ul>

<h2><a id="user-content-config-options" class="anchor" href="#config-options" aria-hidden="true"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1h-1c-1.5 0-3-1.69-3-3.5s1.55-3.5 3-3.5h4c1.45 0 3 1.69 3 3.5 0 1.41-0.91 2.72-2 3.25v-1.16c0.58-0.45 1-1.27 1-2.09 0-1.28-1.02-2.5-2-2.5H4c-0.98 0-2 1.22-2 2.5s1 2.5 2 2.5z m9-3h-1v1h1c1 0 2 1.22 2 2.5s-1.02 2.5-2 2.5H9c-0.98 0-2-1.22-2-2.5 0-0.83 0.42-1.64 1-2.09v-1.16c-1.09 0.53-2 1.84-2 3.25 0 1.81 1.55 3.5 3 3.5h4c1.45 0 3-1.69 3-3.5s-1.5-3.5-3-3.5z"></path></svg></a>Config options</h2>

<ul>
<li><code>escapeMode : true|false</code> - Escaping XML characters. Default is true from v1.1.0+</li>
<li><code>attributePrefix : "&lt;string&gt;"</code> - Prefix for XML attributes in JSon model. Default is "_"</li>
<li><code>arrayAccessForm : "none"|"property"</code> - The array access form (none|property). Use this property if you want X2JS generates additional property _asArray to access in array form for any XML element. Default is none from v1.1.0+</li>
<li><code>emptyNodeForm : "text"|"object"</code> - Handling empty nodes (text|object) mode. When X2JS found empty node like  it will be transformed to test : '' for 'text' mode, or to Object for 'object' mode. Default is 'text'</li>
<li><code>enableToStringFunc : true|false</code> - Enable/disable an auxiliary function in generated JSON objects to print text nodes with text/cdata. Default is true</li>
<li><code>arrayAccessFormPaths : []</code> - Array access paths - use this option to configure paths to XML elements always in "array form". You can configure beforehand paths to all your array elements based on XSD or your knowledge. Every path could be a simple string (like 'parent.child1.child2'), a regex (like /.*.child2/), or a custom function. Default is empty</li>
<li><code>skipEmptyTextNodesForObj : true|false</code> - Skip empty text tags for nodes with children. Default is true.</li>
<li><code>stripWhitespaces : true|false</code> - Strip whitespaces (trimming text nodes). Default is true.</li>
<li><code>datetimeAccessFormPaths : []</code> - DateTime access paths. Use this option to configure paths to XML elements for XML datetime elements. You can configure beforehand paths to all your datetime elements based on XSD or your knowledge. Every path could be a simple string (like 'parent.child1.child2'), a regex (like /.*.child2/), or a custom function. Default is empty.</li>
<li><code>useDoubleQuotes : true|false</code> - Use double quotes for output XML formatting. Default is false.</li>
<li><code>xmlElementsFilter : []</code> - Filter incoming XML elements. You can pass a stringified path (like 'parent.child1.child2'), regexp or function</li>
<li><code>jsonPropertiesFilter : []</code> - Filter JSON properties for output XML. You can pass a stringified path (like 'parent.child1.child2'), regexp or function</li>
<li><code>keepCData : true|false</code> - If this property defined as false and an XML element has only CData node it will be converted to text without additional property "__cdata". Default is false.</li>
</ul>
